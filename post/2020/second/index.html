hugo 搭建博客的完整记录

结合互联网上一些大神的教程，这里记录一下我这个博客搭建的步骤以及踩过的一些坑。

安装Hugo

Hugo 搭建个人博客前，需要先安装 Git 和 Go 语言开发环境。

分别在 https://golang.org/dl 、http://git-scm.com/ 下载自己操作系统对应的安装包，一路默认安装即可。

安装完成后，在cmd输入下面的命令验证是否安装正确。

hugo -v

创建Hugo项目

生成自己的站点:

hugo new site [project-name]

例如我自己的：

hugo new site gaoblog

├── layouts # 存放布局文件
├── static # 存放静态文件 图片 CSS JS文件
└── themes # 存放主题

安装主题

Hugo 有主题市场https://themes.gohugo.io/

安装自己选好的主题，这里我以icar主题为例：

先进入博客的根目录:

cd gaoblog

下载并安装主题:

git clone https://github.com/olOwOlo/hugo-theme-icar themes/icar

启动Hugo本地调试

首先进入 blog/themes/icar/exampleSite 文件夹，将 config.tom 文件拷贝到项目根目录下，同时将 blog/themes/icar/exampleSite/content 文件夹覆盖掉根目录下的 content 。

本地调试:

hugo server

开启服务:

hugo server

浏览器预览，两个地址完全一样:

http://127.0.0.1:1313

http://localhost:1313/

配置主题

这里需要修改 根目录下的config.tom 文件，根据说明修改配置即可。

进入 blog/themes/icar 文件夹，会发现文件结构与新建的 Hugo 项目的文件结构几乎是一样的。这样设置是为了用户的配置可以覆盖掉主题的配置。

比如我要自定义底部的显示，hugo-theme-icar 底部配置由 blog/themes/icar/layouts/partials/footer.html 控制。

为了覆盖掉主题的配置，在项目根目录下新建 blog/layouts/partials/footer.html 文件，填写自定义内容即可覆盖掉主题配置。其他文件的覆盖是一样的。

hugo-theme-icar 使用了 Webpack 打包 js,css,并且文件名加入 hash 值，这种 css,js 是无法覆盖的，不过 hugo-theme-icar 配置中有覆盖这种 css,js 的参数。

创建新的页面

进入生成的site目录：

cd gaoblog

创建一个页面：

hugo new about.md

创建一篇博客

hugo new post/first.md

这个命令会使用模板创建文件，首先查找用户的模板文件，没有就会查找主题的模板文件。

生成Public

确认无误，关闭server服务，将 Markdown 文件打包成 HTML 文件。

hugo -t icar
