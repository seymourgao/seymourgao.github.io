廖雪峰Git教程总结

  1. Git 是用 C语言开发的，创建者是linus大神
  2. 分布式版本控制，不用联网，每个人的电脑都是一台服务器，注意对比集中服务器

安装Git

  1. windows下安装配置，自报家门
1
2

$ git config --global user.name "Your Name"
$ git config --global user.email "email@example.com"
  1. 库/版本库 repository，这个库里的文件都可以被 Git 管理起来

创建版本库

  1. windows建立库
1
2

$ git init
Initialized empty Git repository in /Users/michael/learngit/.git/

初始化一个Git仓库，使用git init命令。

  1. 添加文件到Git仓库，分两步：

  • 使用命令 git add <file>,可以反复多次使用，添加文件；

  • 使用命令 git commit -m <message>,完成。

     没法跟踪Word文件的改动，视频，图片等二进制文件
    
 使用[Notepad++](http://notepad-plus-plus.org/)代替windows记事本

  1. 使用 git status 查看库的状态

    如果`git status`告诉你有文件被修改过，用`git diff`可以查看修改内容。

版本回退

  • HEAD指向的版本就是当前版本，因此，Git允许我们在版本的历史之间穿梭，使用命令git reset --hard commit_id。

  • 穿梭前，用git log可以查看提交历史，以便确定要回退到哪个版本。

  • 要重返未来，用git reflog查看命令历史，以便确定要回到未来的哪个版本。

在Git中，用HEAD表示当前版本，也就是最新的提交1094adb...（注意我的提交ID和你的肯定不一样），上一个版本就是HEAD^，上上一个版本就是HEAD^^，当然往上100个版本写100个^比较容易数不过来，所以写成HEAD~100。

回退上一个版本：

1
2

$ git reset --hard HEAD^
HEAD is now at e475afc add distributed

回到原来版本：

1
2

$ git reset --hard 1094a
HEAD is now at 83b0afe append GPL

1094a这里指版本号，没必要写全，前几位就可以了，Git会自动去找。当然也不能只写前一两位，因为Git可能会找到多个版本号，就无法确定是哪一个了。

工作区和暂存区

  1. .git，这个不算工作区，而是Git的版本库
  2. Git管理的文件分为：工作区，版本库，版本库又分为暂存区stage和暂存区分支master(仓库)

工作区»»暂存区»»仓库

git add把文件从工作区»»暂存区，git commit把文件从暂存区»»仓库，

git diff查看工作区和暂存区差异，

git diff --cached查看暂存区和仓库差异，

git diff HEAD 查看工作区和仓库的差异，

git add的反向命令git checkout，撤销工作区修改，即把暂存区最新版本转移到工作区，

git commit的反向命令git reset HEAD，就是把仓库最新版本转移到暂存区。

撤销修改

1

$ git checkout -- readme.txt

命令git checkout -- readme.txt意思就是，把readme.txt文件在工作区的修改全部撤销，这里有两种情况：

  • readme.txt自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；

  • readme.txt已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。

总之，就是让这个文件回到最近一次git commit或git add时的状态。

本节小结：

1.没有git add时，用git checkout -- file

2.已经git add时，先git reset HEAD 回退到1.，再按1.操作

3.已经git commit时，用git reset回退版本

删除文件

  1. 命令git rm删除一个文件，并且git commit：
1
2
3
4
5
6
7

$ git rm test.txt
rm 'test.txt'

$ git commit -m "remove test.txt"
[master d46f35e] remove test.txt
 1 file changed, 1 deletion(-)
 delete mode 100644 test.txt
  1. 另一种情况是删错了，因为版本库里还有呢，所以可以很轻松地把误删的文件恢复到最新版本：
1

$ git checkout -- test.txt

添加远程库

本地Git仓库和github上的仓库进行同步

在本地的learngit仓库下运行命令：

1

$ git remote add origin git@github.com:michaelliao/learngit.git

本地learngit库就关联github远程库（上面的michaelliao替换成你自己的GitHub账户名）

小结：

  1. 要关联一个远程库，使用命令git remote add origin git@server-name:path/repo-name.git；

  2. 关联后，使用命令git push -u origin master第一次推送master分支的所有内容；

  3. 此后，每次本地提交后，只要有必要，就可以使用命令git push origin master推送最新修改；

从远程库克隆

要克隆一个仓库，首先必须知道仓库的地址，然后使用git clone命令克隆。

Git支持多种协议，包括https，但通过ssh支持的原生git协议速度最快。
